<Heading type="article" level="1">Contributing</Heading>

<Lead>
  Thank you for considering contributing to the Denali Component Library. We
  value your input and efforts to make this a robust and reliable component
  library.
</Lead>

<Heading type="article" level="2">
  Getting Started
</Heading>
Fork the repository and clone it to your local machine. Ensure that you have Node.js
and npm/yarn installed. Follow the README for the initial setup.

<Heading type="article" level="2">
  Contribution Guidelines
</Heading>
Please adhere to the following principles:
<ul class="list-inside list-disc">
  <li>Maintain WCAG 2.1 AA and Section 508 accessibility standards.</li>
  <li>Ensure your components use React hooks.</li>
  <li>Test all components before committing.</li>
</ul>

<Heading type="article" level="2">
  Code Contribution
</Heading>
<ul class="list-inside list-decimal">
  <li>Fork the repo and clone it locally.</li>
  <li>Create a new branch for your feature or bug fix.</li>
  <li>Write code following the project's coding standards.</li>
  <li>Test your code thoroughly.</li>
  <li>Commit and push your changes to your fork.</li>
  <li>Open a pull request against the main branch.</li>
</ul>

<Heading type="article" level="2">
  Figma Documentation
</Heading>
If your contribution involves design elements, please update the Figma documentation
to reflect your changes. Ensure your Figma components match the React components
1:1. See more details in figma file under getting started.

<Heading type="article" level="2">
  Pull Requests
</Heading>
Before submitting a pull request, please ensure:

<ul class="list-inside list-disc">
  <li>All tests pass.</li>
  <li>The code has been reviewed by at least one other person.</li>
  <li>You have updated the Figma documentation if applicable.</li>
  <li>Your commits have clear and descriptive messages.</li>
</ul>

<Heading type="article" level="2">
  Contact
</Heading>
For more information or if you have any questions, feel free to contact us on slack.
